二进制日志

二进制日志介绍

二进制日志的作用是记录数据库里的数据被修改的SQL语句，一般为DDL和DML语句，例如含有insert，update,delete,create,drop,alter等关键字的语句。

二进制日志的作用

1） 记录MySQL的增量数据，用来做增量数据库恢复，没有二进制日志功能，MySQL的备份将无法完整还原数据。
2） 实现主从复制功能

binlog优化思路

1）binlog_cache_szie

二进制日志缓存是数据库为每一个客户连接分配的内存空间，对于事务引擎来说，适当调整该参数会获得更好的性能，参考值如下：

  1. max_binlog_szie

该参数用户设置binlog日志的最大大小，默认为1GB，但是该值并不能严格控制binlog的大小，若binlog大小接近1GB，而此时又在执行一个较大的事务，那么为了保证事务的完整性，数据库不会做日志刷新动作，而是直到该事务的日志全部记录进入当前binlog日志后才会刷新；

3）sync_binlog

该参数的是控制binlog什么时候同步到磁盘；

  • sync_binlog=0 # 表示在事务提交之后，数据库不会将binlog_cache中的数据刷新到磁盘，而是让文件系统 自行决定什么时候来刷新或者缓存满了之后菜刷新到磁盘；

  • sync_binlog=n # 每进行n次事务提交之后，数据库都会进行一次将缓存数据强制刷新到磁盘

设置为0时，数据库性能是最好，但是数据风险也是最大哒

慢查询日志

慢查询日志介绍

慢查询日志就是记录执行时间超出指定值或其他指定条件的SQL语句

慢查询日志参数配置

  • slow-query-log = ON # 慢查询日志开关
  • long_query_time = 2 # 记录大于2秒的SQL语句
  • log_queries_not_using_indexes = ON # 没有使用到索引的SQL语句
  • slow-query-log-file = /application/mysql/data/slow.log # 记录SQL语句的文件
  • min_examined_row_limit = 800 # 记录结果集大于800行的SQL语句

将以上参数写入my.cnf配置文件，重启MySQL即可生效；
